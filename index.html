Brandguide YouRadio

YouRadio ist eine Applikation, in der man sich seinen eigenen “Radiosender” gemeinsam mit anderen Menschen erstellen kann, um mit denen den Konsum von Audio genießen zu können. Die Playlist wird von den Mitgliedern des “Radiosender”-Servers verwaltet. “Radiosender” besser definiert: ein Server, auf dem eine Playlist von Liedern und Sprachnachrichten in Realtime abgespielt wird.

Einleitung

Dies ist der Styleguide für das Web-Projekt YouRadio. Er beinhaltet neben den zu wählenden Farben, auch Beschreibungen der anzuwendenden CSS-Methodologie und deren Class-Naming-Konvention. Des Weiteren werden Code- und styling Beispiele von elementaren Komponenten dargestellt.

CSS-Methodologie

Im Projekt YouRadio wird zum strukturierten Aufbau des CSS BEM angewandt. Was bedeutet das? BEM steht für Block, Element, Modifier. Blöcke sind HTML-Einheiten die unabhängig von ihrem Einsatzort existieren. Beispiele dafür sind "header" oder "footer". Elemente dagegen sind Teile eines Blocks die immer an einen Block gebunden sind. Beispielsweise existiert ein List-Item innerhalb einer Liste. Der letzte Teil des BEM-Konzepts steht für Modifier, das sind "Flags" an Blöcken oder Elementen, die deren Aussehen oder Verhalten ändern. Zum Beispiel "disabled" oder "checked".

Wir verwenden BEM, zum Einen aufgrund der Modularität, denn bei der BEM-Struktur wird nur mit Klassen gearbeitet und die verwendeten Blöcke sind nie voneinander abhängig. Dementsprechend kommt es zu keinen Problemenen mit Cascading und Spezifität. Die Unabhängigkeit der Blöcke macht den Code wiederverwendbar und wartbarer. Zuletzt ist die BEM-Struktur sehr einfach zu verstehen und einzusetzen. Quelle: http://getbem.com/introduction/ , https://de.wikipedia.org/wiki/Block,_Element,_Modifier

Code-Naming-Konventionen

Preprocessor, File-Struktur und CSS-Reset

Im Projekt YouRadio wird SASS/SCSS verwendet. Es wird das 7-1 Pattern angewandt um das SCSS zu strukturieren. Das bedeutet es gibt die Ordner "base, components, layout, pages, themes, abstracts und vendors". Die Files werden dann in einer main.scss gesammelt. Es wird CSS-Normalize angewandt um eine gemeinsame Basis für Styling zu schaffen.

Class-Naming-Konventionen

Klassen werden grundsätzlich klein geschrieben und englisch benannt.

Block

Ein Block ist eine alleinstehendes Entity, die nicht von anderen Elementen abhängt.

Naming HTML CSS
Blöcke werden klein geschrieben. Dabei wird darauf geachtet, dass keine "__" im Namen vorkommen. Jeder DOM-Knoten, der eine Klasse besitzt kann ein Block sein. Wichtig es werden zum stylen nur Klassen vergeben. Die Blöcke werden ausschließlich mit Klassennamen angesprochen. Es ist wichtig, dass beim styling weder CSS, noch Tags verwendet werden.
Beispiel: .block Beispiel: <section class="feed"></section> Beispiel: .feed { color:green;}

Element

Elemente sind Teile von Blöcken, die an Ihren Block gebunden sind.

Naming HTML CSS
Elemente werden klein geschrieben. Der Klassenname wird so gewählt: "Blockname" + "__" + "Elementname" Jeder DOM-Knoten, der eine Klasse besitzt und innerhalb eines Blocks ist kann ein Elementen sein. Wichtig es werden zum stylen nur Klassen vergeben. Die Elemente werden nur einzeln und direkt per Klasse angesprochen. So entsteht keine Hierarchie.
Beispiel: .feed__post Beispiel: <section class="feed"><div class="feed__post"></div></section> Beispiel: .feed__post { color:green;}

Modifier

Modifier sind Flags an Elementn oder Blöcken, die deren Aussehen oder Zustand verändern.

Naming HTML CSS
Modifier werden klein geschrieben. Der Klassenname wird so gewählt: "Blockname" + "__" + "Elementname" + "--" + "Modifiername" oder "Blockname" + "--" + "Modifiername". Modifier sind extra Klassennamen die zu jedem Block oder Element hinzugefügt werden können. Modifier werden einzeln und direkt per Klasse angesprochen. So entsteht keine Hierarchie.
Beispiel: .feed__post--color-blue Beispiel: <section class="feed"><div class="feed__post--color-blue"></div></section> Beispiel: .feed__post--color-blue { color:green;}
